# Security policy

## Supported versions

Security fixes are applied to the latest release and the current default branch. Older releases are not supported unless a maintainer states otherwise in a security advisory.

## Report a vulnerability

Do not disclose suspected vulnerabilities in a public issue, discussion, or pull request. Use GitHub's [private vulnerability reporting](https://github.com/adamaoc/photopti-app/security/advisories/new) to send the affected version, impact, reproduction steps, and any suggested mitigation. If that form is unavailable, email [hello@ampnet.media](mailto:hello@ampnet.media) with the subject `Photopti security report`.

You should receive an acknowledgement within seven days. The maintainer will investigate, coordinate a fix and release when warranted, and credit reporters who want attribution. Please allow a reasonable remediation period before public disclosure.

Never include real personal images, credentials, signing materials, or other sensitive data in a report. Use a minimal synthetic reproduction.
